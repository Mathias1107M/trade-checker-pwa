# 交易檢查工具 README

這是一個純前端的 PWA（Progressive Web App）交易檢查工具。

用途是讓你在交易前逐題回答問題，系統會將你的回答與預設答案比對：

- 全部正確：顯示「`✅ 檢查通過，可以交易`」
- 任一題錯誤：顯示「`❌ 檢查未通過，請冷靜 10 分鐘`」，並鎖定 10 分鐘

本專案不需要後端，不需要資料庫，使用 `HTML + CSS + JavaScript` 即可運行。

## 檔案說明

本專案包含以下五個主要檔案：

### 1. `index.html`

主頁面檔案。

用途：

- 建立網頁基本結構
- 載入 `style.css`
- 載入 `app.js`
- 設定 PWA 需要的 `manifest.json`

你可以直接用瀏覽器打開這個檔案來使用基本功能。

### 2. `style.css`

畫面樣式檔案。

用途：

- 定義手機優先（mobile-first）的版面
- 設定按鈕、卡片、顏色、進度條
- 顯示成功時的綠色畫面
- 顯示失敗鎖定時的紅色畫面

### 3. `app.js`

主要邏輯檔案。

用途：

- 儲存題目資料
- 顯示每一題問題
- 處理「是 / 否」回答
- 驗證答案是否符合 `correctAnswer`
- 顯示進度
- 若回答錯誤，使用 `localStorage` 鎖定 10 分鐘
- 若全部正確，顯示可交易結果

### 4. `manifest.json`

PWA 設定檔。

用途：

- 定義 App 名稱
- 設定主題色與背景色
- 讓網站可加入手機主畫面
- 讓 PWA 以接近 App 的方式開啟

### 5. `service-worker.js`

Service Worker 檔案。

用途：

- 快取網站需要的檔案
- 支援離線使用
- 提升再次開啟時的載入速度

## 如何使用

### 方式一：直接打開 `index.html`

最簡單的方式：

1. 確認以下檔案放在同一個資料夾中：
   - `index.html`
   - `style.css`
   - `app.js`
   - `manifest.json`
   - `service-worker.js`
2. 用瀏覽器直接打開 `index.html`
3. 開始逐題回答問題

這種方式可以使用：

- 題目顯示
- 是 / 否作答
- 正確 / 錯誤判斷
- 10 分鐘鎖定
- `localStorage` 保存鎖定時間

注意：

- 若是用 `file://` 直接開啟，通常主功能沒問題
- 但某些瀏覽器不會在 `file://` 模式下啟用完整 PWA 功能
- 例如 Service Worker、加入主畫面、完整離線快取，通常需要透過本機伺服器開啟

## 若要完整啟用 PWA 功能

如果你要使用以下功能：

- 加入主畫面
- 離線運行
- Service Worker 快取

建議使用本機靜態伺服器來開啟專案。

### 方法範例：使用 VS Code Live Server

1. 用 VS Code 開啟這個專案資料夾
2. 安裝 `Live Server` 擴充套件
3. 在 `index.html` 按右鍵
4. 選擇 `Open with Live Server`
5. 用瀏覽器開啟顯示的網址

之後你就可以測試：

- PWA 安裝
- 離線快取
- 手機加入主畫面

### 方法範例：使用 Python 簡易伺服器

如果你的電腦有安裝 Python，也可以在專案資料夾執行：

```bash
python -m http.server 8000
```

然後在瀏覽器打開：

```text
http://localhost:8000
```

## 手機使用方式

### Android（Chrome）

1. 用本機伺服器開啟這個網站
2. 在 Chrome 中打開該網址
3. 點選瀏覽器選單
4. 選擇「加入主畫面」或「安裝 App」

### iPhone / iPad（Safari）

1. 用本機伺服器開啟這個網站
2. 在 Safari 中打開該網址
3. 點選分享按鈕
4. 選擇「加入主畫面」

## 使用流程

1. 打開工具
2. 系統從第 1 題開始顯示
3. 每題只能回答一次，逐題往下
4. 當前題目回答後才會進入下一題
5. 如果所有題目都符合預設答案：
   - 顯示 `✅ 檢查通過，可以交易`
   - 畫面變成綠色
6. 如果任何一題不符合預設答案：
   - 顯示 `❌ 檢查未通過，請冷靜 10 分鐘`
   - 畫面變成紅色
   - 系統鎖定 10 分鐘
   - 期間不能重新測試

## 鎖定機制說明

系統使用瀏覽器的 `localStorage` 記錄鎖定時間。

這代表：

- 即使你重新整理頁面，鎖定仍然有效
- 關閉瀏覽器後重新打開，只要 10 分鐘還沒到，仍然會保持鎖定
- 10 分鐘到期後，系統會自動恢復可測試狀態

注意：

- 如果手動清除瀏覽器網站資料或 `localStorage`，鎖定紀錄可能會消失

## 檔案放置規則

請務必讓這五個檔案位於同一個資料夾：

```text
index.html
style.css
app.js
manifest.json
service-worker.js
```

如果檔案位置錯誤，可能會導致：

- 樣式沒有載入
- JavaScript 沒有執行
- PWA 功能失效
- 無法離線使用

## 自訂修改

如果你想自行調整內容，可參考以下位置：

- 修改題目：
  編輯 `app.js` 內的 `questions`
- 修改顏色與外觀：
  編輯 `style.css`
- 修改 App 名稱與 PWA 設定：
  編輯 `manifest.json`
- 修改快取策略：
  編輯 `service-worker.js`

## 常見問題

### 為什麼直接打開 `index.html` 可以用，但不能安裝成 App？

因為大多數瀏覽器對 PWA 有安全限制。
`Service Worker` 通常需要在：

- `localhost`
- `http://127.0.0.1`
- 或正式網站的 `https`

環境下才會正常運作。

### 為什麼我重新整理後還是被鎖住？

因為鎖定時間是存在 `localStorage` 中，這是設計上的正常行為。

### 可以改成更多題嗎？

可以，直接在 `app.js` 的 `questions` 陣列中增加題目即可。

## 總結

如果你只想快速使用：

- 把五個檔案放在同一資料夾
- 直接打開 `index.html`

如果你想完整使用 PWA：

- 用本機伺服器開啟
- 再用手機或電腦瀏覽器安裝到主畫面
